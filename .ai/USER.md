# User Notes

## Preferences

- Communicate in Vietnamese unless the code/task context is better served in English.
- Work step by step.
- For code tasks, execute the full lifecycle: analyze codebase if needed, analyze requirements, plan, implement, design test cases, review code, run tests, run build/lint, and delete temporary test-case files before finishing.
- Before substantial work, state the next step briefly.
- Automatically load relevant skills when the task matches a skill.
- Store important project, user, and collaboration notes under `.ai/`.
- Prefer practical implementation over lengthy proposals when the request clearly asks for changes.

## Collaboration Style

- Be direct and factual.
- Keep final responses concise, listing changed files and validation results.
- Ask a short clarification only when a decision affects behavior or compatibility and cannot be inferred safely.
- Report validation honestly; do not claim checks that were not run.
- Do not commit unless explicitly requested.
